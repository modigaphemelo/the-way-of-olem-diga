# The Way of Olem Diga

A personal manual for a specific mind in a specific world.

**INTP cognitive architecture. Enneagram 5w4. Level 1 ASD sensory and executive profile.**

---

## What This Is

This is not a blog. Not a guide for the general public. It is a **personal operating system**—a set of protocols, thresholds, and practices designed for one person to navigate a world not built for his wiring.

The site documents a four-stage sequential framework:

| Stage | Focus | Core Question |
| :-- | :-- | :-- |
| Ⅰ · Foundation | Sovereignty & Self-Knowledge | Can I sustain myself? |
| Ⅱ · Depth | Shadow Integration & Resilience | What am I capable of? |
| Ⅲ · Construction | Building & Creating | Am I building something that matters? |
| Ⅳ · Legacy | Contribution & Exit | What remains when I'm gone? |

Each stage contains modules with specific practices. Between stages are **threshold checks**—falsifiable gates that must be passed before proceeding. This is not a buffet. The stages are sequential.

---

## Why This Exists

Standard advice assumes a different operating system. It assumes real-time processing, unlimited social energy, and functional executive ignition. That advice failed repeatedly.

This manual is the result of documenting what *actually* worked—not what should have worked, not what sounded good. It is a living document, updated as new data arrives.

## Technical

This is a static HTML site with no build process, no framework, and no dependencies beyond a CSS file and a few lines of vanilla JavaScript.

```

/
├── index.html          # Landing page with four-stage overview
├── begin.html          # Origin story and why this exists
├── foundation/         # Stage Ⅰ modules (3 files)
├── depth/              # Stage Ⅱ modules (3 files)
├── construction/       # Stage Ⅲ modules (3 files)
├── legacy/             # Stage Ⅳ modules (3 files)
├── threshold/          # 5 threshold checks
├── css/way.css         # Single stylesheet (dark theme, responsive)
├── js/script.js        # Minimal JS (sidebar toggle, progress bar)
└── README.md

```

The design is intentionally minimal. Dark background, single warm accent, no distractions.
